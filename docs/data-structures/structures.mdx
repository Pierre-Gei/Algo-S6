## Structures de Données

Dans notre application, nous utilisons plusieurs structures de données pour gérer les règles, les conditions, les faits et les réponses. Voici une description de chaque structure :

### Liste des Règles (`liste_regles`)

Cette structure représente une liste chaînée de règles. Chaque élément de la liste contient :

- `char *name` : Le nom de la règle.
- `struct liste_conditions *conditions` : Une liste chaînée des conditions de la règle.
- `struct liste_regles *suivant` : Un pointeur vers la règle suivante dans la liste.

---

### Liste des Conditions (`liste_conditions`)

Cette structure représente une liste chaînée de conditions. Chaque élément de la liste contient :

- `char *name` : Le nom de la condition.
- `struct liste_conditions *suivant` : Un pointeur vers la condition suivante dans la liste.

---

### Liste des Faits (`liste_faits`)

Cette structure représente une liste chaînée de faits. Chaque élément de la liste contient :

- `char *fait` : Le fait lui-même.
- `struct liste_faits *suivant` : Un pointeur vers le fait suivant dans la liste.

---

### Liste des Réponses (`liste_reponses`)

Cette structure représente une liste chaînée de réponses. Chaque élément de la liste contient :

- `char *name` : Le nom de la réponse.
- `struct liste_reponses *suivant` : Un pointeur vers la réponse suivante dans la liste.

Ces structures de données sont essentielles pour le fonctionnement de notre application. Elles permettent de stocker et de manipuler les règles, les conditions, les faits et les réponses de manière efficace.
